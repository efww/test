# AIP PDF 읽기 전용 관문 PoC 협의 요청서

## 요청 요약

Foxit 기반 AIP PDF 열람 안정성과 사용성 문제가 반복되어, 공식 Microsoft 권한 체계를 우회하지 않는 읽기 전용 관문 뷰어 PoC를 제안드립니다.

이번 PoC는 보호 해제 도구가 아닙니다. 사용자의 기존 Microsoft Purview/Azure Information Protection 권한을 확인하고, 권한이 있는 문서만 읽기 전용 화면에 표시하는 가능성을 검증하는 제한 PoC입니다.

## 1차 기술검증 결과

회사 AIP 문서에 접근하지 않는 선에서 다음을 구현하고 검증했습니다.

- 일반 PDF 열람 화면
- PDF.js 기반 메모리 바이트 렌더링
- 저장, 인쇄, 다운로드, 복사 버튼 미노출
- `Ctrl+P`, `Ctrl+S`, `Ctrl+C`, 우클릭, 드래그 차단 골격
- Microsoft 로그인 진입점과 앱 승인 정보 필요 상태
- 보호 PDF 의심 파일 조기 차단 자리
- 로컬 JSONL 감사 로그

## 회사 측 요청사항

2차 PoC 진행을 위해 아래 항목이 필요합니다.

- Entra ID 앱 등록 또는 관리자 승인
- 테스트 사용자 3~5명 지정
- 테스트용 AIP PDF 3~5개 제공
- 저장, 복사, 인쇄, 텍스트 추출, 일반 PDF export 금지 조건 승인
- 감사 로그 항목과 보관 위치 합의
- PoC 기간과 테스트 환경 승인

## 2차 PoC 범위

2차에서는 회사 승인 범위 안에서만 실제 AIP PDF를 다룹니다.

- 공식 MIP SDK 연동
- 승인된 테스트 사용자 로그인
- 테스트 AIP PDF 열람 권한 확인
- 권한 있는 문서만 읽기 전용 표시
- 권한 없는 문서, 만료 문서, 정책 오류 차단
- 평문 PDF 잔존 파일 검사
- 감사 로그 검증

## 기본 금지사항

다음은 PoC 범위에서 제외합니다.

- AIP 보호 해제
- 일반 PDF export
- 서버 기반 대량 복호화
- PDF 편집
- 전사 배포
- Foxit 전체 기능 대체
- 화면 캡처 완전 차단 같은 전문 DRM 기능

## 보안 설명

PoC 앱은 Microsoft 권한 체계를 우회하지 않습니다. 실제 AIP PDF 처리는 회사 승인 후 공식 MIP SDK를 통해서만 진행합니다. 1차 앱은 보호 PDF 의심 파일을 열지 않고 차단하도록 설계되어 있습니다.

복호화된 PDF를 일반 파일로 남기지 않는 구조를 목표로 하며, 2차에서는 앱 종료 후 temp/cache/log/downloads 경로에 평문 PDF나 문서 본문이 남지 않는지 실물 테스트로 확인합니다.

## 성공 기준

2차 PoC는 아래 조건을 충족해야 성공으로 봅니다.

- 권한 있는 테스트 사용자는 지정 AIP PDF를 열람할 수 있습니다.
- 권한 없는 사용자는 문서를 볼 수 없습니다.
- 저장, 인쇄, 복사, 다운로드, 텍스트 추출이 기본 차단됩니다.
- 로그에 사용자, 문서 식별자, 결과, 시각이 남습니다.
- 로그에 토큰, PDF 본문, 복호화 산출물이 남지 않습니다.
- Foxit 문제 문서에서 열람 안정성 또는 사용성 개선이 확인됩니다.
